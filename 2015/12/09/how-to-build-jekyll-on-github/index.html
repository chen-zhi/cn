如何用Jekyll在GitHub搭建免费的，无限流量的博客

发表于 2015-12-09   |  

背景

作为一个在芬兰学校计算机的学生，出于对Linus的崇拜，我决定学习Git并写独立博客。独立博客可用Jekyll, Octopress, hexo, pelican 等搭建，此文讲述Jekyll在Mac下的使用。

Jekyll 究竟是什么？

下文摘取自官网介绍

Jekyll 是一个简单的博客形态的静态站点生产机器。它有一个模版目录，其中包含原始文本格式的文档，通过一个转换器（如 Markdown）和我们的 Liquid 渲染器转化成一个完整的可发布的静态网站，你可以发布在任何你喜爱的服务器上。Jekyll 也可以运行在 GitHub Page 上，也就是说，你可以使用 GitHub 的服务来搭建你的项目页面、博客或者网站，而且是完全免费的。

本教程适用人群

  • Mac拥有者
  • 了解基本建站知识
  • 会使用基本的Terminal命令
  • 了解基本Git操作

备注：什么都不会，但是对独立博客感兴趣的人也可以试试哦。搭建过程并不难。

搭建Jekyll的准备

  • Markdown编辑器(本人使用Mou)
  • 域名购买（本人选择 Godaddy）
  • Web编辑器（本人使用Brackets, Sublime和Atom也是不错的选择）

基本步骤

Step 1: 注册GitHub

没有GitHub账号？前往https://github.com/注册。

登陆GitHub,新建一个repository。命名为username.github.io［请把username改成自己的用户名］

Step 2: 域名购买，绑定自定义域名

  1. 域名购买,推荐Godadday -> 网址https://www.godaddy.com/
  2. 购买域名完毕后返回刚刚新建的username.github.io界面。选择New file, 命名为CNAME（没有后缀名）, 在空白处输入自己的刚刚买好的域名。如下图所示。
  3. 返回Godaddy -> 在My Domian的设置里找到Manage DNS -> 删除DNS ZONE FILE里所有的配置。
  4. 增加一条A记录(“host” = @ 和 “Points to” = 192.30.252.153)
  5. 增加一条CNAME记录（”host” = www 和”Points to” = username.github.io）
  6. 在浏览器里打开username.github.io会自动跳转到你自己的域名。如果出现404，不要着急喝杯咖啡稍等几分钟重试。

Step 3: 安装git

打开Terminal输入git，看看系统有没有安装Git（本人在此之前就已装）

如果没有安装怎么办呢？很简单，打开App Store, 搜索Xcode并安装。运行Xcode，找到“Xcode”菜单->“Preferences”->“Downloads”->“Command Line Tools”，安装。

Step 4: 安装Jekyll

  1. 安装Ruby, 大多数Mac自带Ruby. 查看版本号ruby --version。 需要版本号在2.0.0以上。
  2. 安装Bundler。gem install bundler
  3. 安装NodeJS: NodeJS官网，查看版本号node -v
  4. 安装Jekyll:

注意：许多在terminal执行的命令前需要加上sudo

Step 5: 其他

1.建立gh-pages分支

建立一个没有父节点的分支gh-pages。github规定，在项目类型的仓库中，只有该分支中的页面，才会生成网页文件

2.配置ignored files文件

打开GitHub DeskTop,在菜单栏找到repository->repository setting->ignored files 添加_side。

在启动Jekyll server后，Jekyll会自动创立_side文件夹，并把转化的静态页面保存在里面。

3.模板

前往http://jekyllthemes.org/去寻找心仪的模板吧。不过Jekyll模板真的不好用！！！
